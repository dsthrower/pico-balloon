# Pico Balloon Project — KK7OWL

20m WSPR pico balloon build (Traquito Jetpack). This repo lives in iCloud Drive.

## Sync workflow
When I say "sync" or "save my work", do all of the following in order:
1. Run `git fetch` then `git status` to report where local and origin/master stand.
2. If there are remote changes, `git pull` first.
3. If there are local changes, stage everything (`git add -A`), commit with a
   clear message describing what changed, and `git push`.
4. Report the final state in one or two plain sentences.

Always show me the commit message before pushing, and pause if there is a
merge conflict instead of forcing anything.
